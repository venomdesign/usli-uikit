Input Component

A flexible text input component with built-in validation, error handling, and support for various input types.

Validation Examples

The Input component supports reactive forms validation with real-time feedback.

Email Validation

<app-usli-input
  type="email"
  placeholder="Enter your email"
  label="Email Address">
</app-usli-input>

Required Field

<app-usli-input
  placeholder="This field is required"
  label="Full Name"
  required="true">
</app-usli-input>

Min/Max Length

<app-usli-input
  placeholder="Enter 3-20 characters"
  label="Username"
  minlength="3"
  maxlength="20">
</app-usli-input>

Form Integration Patterns

Integrate the Input component with Angular Reactive Forms for advanced form management.

FormGroup Integration

// TypeScript
export class MyComponent {{
  form = new FormGroup({{
    email: new FormControl('', [Validators.required, Validators.email])
  }});
}}

// HTML
<app-usli-input
  label="Form Field with FormControl"
  placeholder="Bound to FormControl">
</app-usli-input>

Two-Way Binding

<app-usli-input
  placeholder="Type to see value update"
  label="Two-Way Bound Field"
  [(ngModel)]="fieldValue">
</app-usli-input>

API Documentation

Complete reference for Input component inputs, outputs, and properties.

Inputs

Property Type Default Description
type string 'text' Input type (text, email, password, number, tel, url, search, etc.)
placeholder string undefined Placeholder text displayed when input is empty
label string undefined Associated label text for the input field
value string '' Initial value of the input
disabled boolean false Disables the input field
required boolean false Marks the input as required
readonly boolean false Makes the input read-only
minlength number undefined Minimum character length validation
maxlength number undefined Maximum character length validation
pattern string undefined Regular expression pattern for validation
error boolean false Displays error state styling
success boolean false Displays success state styling
hint string undefined Helper text displayed below the input
icon string undefined Icon name or class to display inside input
clearable boolean false Shows a clear button to reset the input

Outputs

Event Type Description
valueChange EventEmitter<string> Emits when the input value changes
focus EventEmitter<FocusEvent> Emits when the input receives focus
blur EventEmitter<FocusEvent> Emits when the input loses focus
keydown EventEmitter<KeyboardEvent> Emits when a key is pressed
keyup EventEmitter<KeyboardEvent> Emits when a key is released
clear EventEmitter<void> Emits when the clear button is clicked
